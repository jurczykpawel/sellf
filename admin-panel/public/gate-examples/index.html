Sellf element gating — interactive demo

Gate parts of a page and show different content per visitor. Pick a visitor below and watch every block react — this page is self-contained (no backend) so you can see all the states. See the real snippet to deploy →

1 · Three states, one block

Wrap a block in data-sellf-product="<slug>" and add up to three branches. Exactly one is shown.

3 · Gate a feature (in-browser work)

Add data-sellf-feature="<slug>" to a control. Owners get it enabled; everyone else gets it locked. Best for work that runs locally (e.g. rendering captions in the browser).

Render captions — 🔒 locked✓ enabled

4 · Gate a real backend action

In-browser locks are best-effort. For an action that costs money or runs on a server, confirm access with SellfGate.verify(slug) first — the server checks the signed token, so it can’t be faked from the browser.

What SellfGate.verify('pro-kit') returns for this visitor:

✅ verified server-side — run the paid action 🔒 denied — show the upgrade dialog instead 

if (await SellfGate.verify('pro-kit')) {
  runExpensiveThing();        // server already confirmed ownership
} else {
  showUpgradeDialog();
}

The snippet behind all of this

In production you set no state yourself — you paste one snippet and Sellf adds the right state to each block from the visitor’s signed access token. Full deploy-ready example →

<!-- in <head> -->
<script src="https://YOUR-SELLF.example/api/loginwall/gate.js?products=pro-kit,vip-masterclass"></script>
<script>!window._SF_GATE_XXXX && (location.href =
  "https://YOUR-SELLF.example/loginwall/gate?products=pro-kit,vip-masterclass&redirect=" +
  encodeURIComponent(location.href));</script>

This demo switches visitors with pure CSS so you can explore every state. In production the runtime resolves each visitor from their signed token and removes the inactive branches from the page. Note: all branch HTML still ships in the page source, so display gating is best-effort UX — protect anything sensitive or paid server-side via SellfGate.verify() (section 4), never by display state alone.
